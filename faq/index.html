FAQ

The questions that decide whether the tool is worth using.

Is this a SaaS product?
No. It is a local-first CLI and a public landing site. The tool runs on extension folders, not on hosted user accounts.
Does it upload my code?
Not by default. The MVP is designed to run locally and produce files and reports on your machine.
Does it fix every MV2 extension automatically?
No. It handles the common migration path and keeps unsafe or ambiguous cases visible as warnings.
What should I validate after running it?
Load the output as an unpacked extension in Chrome, check the report, and review any webRequest or background-worker warnings.
What has already been tested?
Free runs, signed token issuance, token-based paid unlock, batch mode, patch export, DNR drafts, and a Chrome unpacked smoke script.

mv3migrate

Chrome MV2 to MV3 migration CLI. It rewrites the common parts, flags the dangerous parts, and keeps the output readable enough for a real review.
